Chapter 6 Terms
Term Definition

Machine Language
  • A computer programming language consisting of binary or hexadecimal instructions which a computer can respond to directly
    		•

    Debugging
  • the process of finding errors that must be located and then corrected

    • Assembler
  • programs developed to convert mnemonic experssions into machine language instructions.

    • Assembly Language
  • a mnemonic system for representing programs
    		•

    Translator
  • a program that takes a collection of high-level Primitive languages and converts them into machine-language programs.

    • Compiler
  • a program that simulates the activity of a single high-level primitive language.

    • Interpreter
  • in the third generation of languages that executed instructions as they were translated instaed of recording the translated version for future use.

    • Cross-platform Software
  • software that is independent of an operating system's design and the machine's hardware design and is executable throughout a network.

    • Natural Language
  • language that is precisely defined by grammars (e.g. english, german, latin, etc.)

    • Formal Language
  • language that evolved over time without formal grammatical analysis
    		•

    Machine independence
  • software that could be transported from one machine to another with relative ease.

    • High-Level programming language
  • languages which provvide more abstractions for the programmer and make it easier for people to read and write a program. (python vs assembly)

    • object-oriented programming (OOP)
  • the approach to programming where software design is around data rather than function and logic.

    • Scripting language
  • languages typically used to perform administrative tasks. (PERL/PHP for instance)
    		•

    data types:
  • integer

    • numeric data consisting of whole numbers.

  • float

    • numeric data that might contain values other than whole numbers.

  • string

    • sequence of characters.

    Primitive data type
  • set of basic data types from which all other data types are constructed
    		•

    Data structure
  • the conceptual shape or arrangement of data.
    		•

    Python collection data structures
  • list

    • used to store multiple items in a single variable.

  • tuple

    • A collection which is ordered and unchangeable round brackets "()"

  • set

    • A collection which is unordered, unchangeable, and unindexed. *but can remove and add items*

    function
  • a procedure without defining a returned value
    		•

    Local Variable
  • A variable declared within a function
    		•

    global variable
  • variables not restricted to a particular part of a program
    		•

    Source program
  • A program in its original form and original language.
    		•

    Object program
  • The translated version of the source program.
    		•
