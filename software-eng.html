Chapter 7: Software Engineering
Term Definition

software engineering
  • The branch of computer science that seeks principles to guide the development of large, complex software systems.
    		•

    Integrated development environment (IDE)
  • Software that has combined development software (editors, compilers, debugging tools, etc.)
    		•

    Association for Computing Machinery (ACM)
  • Purpose of ACM
      • An international scientific and educational organization dedicated to advancing the arts, sciences, and applications of information technology.
  • Benefits of ACM membership
      • Connect to peers in the ACM and learn from their resources
  • student ACM membership
      • You get networking opportunities including a job center, the opportunity to join the special interest groups in ACM.

    Institute of Electrical and Electronics Engineers (IEEE)
  • Purpose of IEEE
      • An organization of electrical, electronics, and manufacturing engineers.
  • benefits of IEEE membership
      • Network with professionals within your local areas, Gain access to changes in technology and access resources and opportunities within IEEE etc.
  • student IEEE membership
      • networking and workshopping opportunities but must be within the IEEE designated fields.

    Software Life Cycle:Development phase
  • Requirements analysis
      • Analysis on specifications a proposed system will provide and to identify any conditions on those services and to define how the outside world will interact with the system.
  • Design
      • creation of a plan for the construction of the proposed system. - basically developing a solution to the problem.
  • Implementation
      • The actual writing of programs, creation of data files, and development of databases
  • Testing
      • formerly known for debegging and final confirmation of compatibility in software requirements.

    modular implementation
  • The division of software into manageable units.
    		•

    Unified Modeling Language (UML)
  • A collection of tools with object-oriented programming in mind, basically classes and relationships between classes.
    		•

    use case
  • a proposed system of interactions between systems and it users.
    		•

    use case diagram
  • a visual representation of interactions between a system and its users
    		•

    Data relationships:
  • one-to-one
      • the association between patients and occupied private rooms in that each patient is associated with only one room and each room is accociated with only one patient.
  • one-to-many
      • The association between physicians and patients in that one physician is associated with many patients and eath patient is associated with one (primary) physician
  • many-to-many
      • including consulting physicians in the physician-patient relationship - then each physician could be associated with many patients and each patient could be associated with many physicians.

    design pattern
  • a predeveloped model for solving a recurring problem in software design.
    		•

    Software quality assurance (SQA)
  • a group that is in charge of overseeing and enforcing the quality control systems adopted by the organization. - approving software requirements and specifications before the design stage.
    		•

    Software testing:
  • glass box testing
      • The software tester is aware of the interior structure of the software and uses this knowledge when designing the test.
  • black box testing
      • Tests that do not rely on knowledge of the software's interior composition.
  • beta testing
      • A preliminary version of the software that is given to a segment of an intended audience with the goal of learning how the software performs in real-life situations.
  • alpha testing
      • A testing environment that takes place within the organization.

    Documentation:
  • user documentation
      • information that is used to explain the features of the software and describe how to use them
  • system documentation
      • information on the software's internal composition so that the software can be maintained later in its life cycle.
  • technical documentation
      • Information that describes how a software system should be installed and serviced.

    Intellectual property law
  • software license
      • a legal agreement between the owner and user of a software product taht grants the user certain permissions to use the product without transferring ownership rights to the intellectual property.
  • software disclaimer
      • a statement on limitations for liability on the operations of the software.
  • best defenses against software liability claims
      • Apply sound software engineering principles during the software's devvelopment, to use a level of care compatible with the software's application, and to produce and maintain records that validate these endeavors.
