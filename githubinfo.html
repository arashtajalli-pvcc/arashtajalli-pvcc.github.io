All about Github
Q. What is a version control system? A. the practice of tracking and managing changes to software code.
Q. What is Git? A. A distributed version control system that tracks changes in any set of computer files.
Q. What is Github? A. Cloud based service for software development.
Q. What is the difference between Git and Github? A. Github is a cloud hosting service that lets you manage git repositories, while git is a software service on a computer to keep track of code history.
Q. Who started Github and how was it started? A. Chris Wanstrath and PJ Hyett, both founders were making improvements to ruby on rails but in the process of updating code they found that they had to lobby to commit any changes to the central code and decided to make their own repository.
Q. What company owns it now? A. Microsoft
Q. How much does a Github account cost? A. There are tiers a free account, team account at $4.00 per user/month, and Enterprise at $21.00 per user/month.
Q. What is the Octocat? A. the mascot of Github, perhaps a reference to the show Spliced (TV Series)
Git/Github terms:
repository - (repo) a type of centrally located storage for project files and resources.
commit - revision, an individual change to a file.
fork - a new repository that shares code and visibility settings with the original "upstream" repository.
push - used to upload local repository contents to a remote repository.
pull requests - when a developer is ready to begin the process of merging new code changes with the main project repository.
workflows - a configurable automated process that will run one or more jobs.
issues - a description of a bug or feature, used in github to keep track of the issue at hand.
raw button - view the copy or raw content of a file without any styling.
blame button - annotates each line in a given file to find who made "changes that made a bug"
